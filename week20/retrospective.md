# 振り返りレポート：Week20

## 学んだこと

### アジャイル開発
- ウォーターフォールと違い、短いサイクルで繰り返し開発することを学んだ
- 計画を固めすぎず、変化に柔軟に対応することが大切だとわかった

### スクラム
- Sprint・デイリースクラム・レトロスペクティブの流れを理解した
- タスクをユーザーストーリーで表現することで、誰のための機能かが明確になった

### カンバンボード
- GitHub ProjectsでTodo・In Progress・Review・Doneの4列を作成した
- WIP制限（In Progressは3つまで）で集中して開発できることがわかった

### KPI設定
- 「毎日使い続けてほしい」「無駄遣いを減らしてほしい」という目標を
  DAU・節約目標達成率・レシート撮影利用率という数値に落とし込めた
- 曖昧な目標を数値化することで、達成できたかどうか判断できるようになる

### 見積書
- 工数から単価を計算して見積書を作成する流れを理解した
- バッファ30%を含めることでリスクに備えることを学んだ

## 難しかったこと
- ユーザーストーリーの優先順位付けが難しかった
- KPIの指標が似たものになりがちで、視点を変える必要があった
